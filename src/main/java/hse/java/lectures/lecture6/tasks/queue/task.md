# Ограниченная Блокирующая Очередь (`BoundedBlockingQueue`)

`CI-тег коммита:` `queue:`

## Условие
Реализовать потокобезопасную ограниченную очередь `BoundedBlockingQueue<T>` 
follow up - без `java.util.concurrent`.

Очередь должна корректно работать при нескольких producer и consumer потоках.

## Требования
1. `capacity > 0`, иначе `IllegalArgumentException`.
1. `null` в `put` запрещен (`NullPointerException` или `IllegalArgumentException`).
1. `put` блокируется, если очередь полная.
1. `take` блокируется, если очередь пустая.
1. FIFO должен соблюдаться.
1. Синхронизация только через `synchronized`, `wait`, `notifyAll`.
1. Нельзя использовать `java.util.concurrent`.
1. Очередь должна корректно себя вести при прерываниях потоков писателей и читателей

## Как работает класс
Если очередь пуста, при получении элемента из очереди потоки читатели ждут, то есть получить элемент из пустой очереди - блокирующая операция, 
при заиси элемента в очередь свыше лимита(вместимости очереди), потоки писатели ждут, это тоже блокирующая операция. Таким образом у нас остается труба в котороую можно записать параллельно только N сообщений, но при чтении сообщения из очереди пропадают. 
То есть по другому эту задачу можно назвать - сделать буфферизированный Go-шный канал
